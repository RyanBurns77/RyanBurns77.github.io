Blog Post #2 (2/17/23)

Progress towards the final design of the project is going somewhat smooth so far.  We are currently in the beginning stages of trying to implement machine learning into the project, and we appear to be on pace to finish the Alpha stage of the project on time, as indicated by the Gantt chart from the previous blog post.  For the final design document, something else that I brought up in the previous blog post was the structure of our project, and how the later sections of the design document would probably not be able to be fully completed until the project itself is nearing completion.  So, there is not much progress to report in terms of the final design document at this point, but I expect this to change as we begin wrapping up the first part of the project and moving onto the Beta stage, where screen-reading will be implemented.

Since we have been using a version of Connect 4 created in Python as the base for the project, something that has been implemented is the process of outputting the game state after every move is made.  This will need to be changed slightly in the future to better match the format of the dataset that we plan on using for training the machine learning algorithm, but for now it does what it is supposed to do.  There has also been some experimentation with the AI that is already present in the Python Connect 4 code, and in an effort to better understand the board scoring system that is used and how to improve it, we have the AI playing against itself in the current version.  Since I have been tasked with improving said scoring system that our AI will ultimately use, this is helpful when trying to understand why certain moves are happening, and how to better reward certain actions over others.  This will also help when testing the accuracy of the game state output when it gets reformatted.

Although many of the concepts involved in this project are still quite new to me, I haven’t run into anything too challenging so far.  This will likely change (if only slightly) as I begin working on implementations, especially for something like the scoring system used for game states.  However, I am hopeful that I will be able to use what I already know, and what I have learned this semester to get everything finished.  Another challenge has been deciding on meeting times and formats, separate from class time on Monday and our scheduled one-on-one meetings on Thursday.  We have typically been having brief check-in meetings over the weekend, but I would like to see this shift into a longer, more work-oriented type of meeting, as I think it would be very helpful if we took some time to sit down and look over the code together.  I’ve always found it helpful to look through code and go over ideas with someone else, so even though our current meeting structure seems to be working, I hope we can have them be more useful going forward.  Other than that, I think that the group as a whole has been making good progress, and I think I have a good understanding of what I need to revise and implement as we approach the end of the Alpha stage of the project.
