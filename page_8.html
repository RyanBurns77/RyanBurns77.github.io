Blog Post #5 (4/14/23)

At this point in the semester, we are bringing work on the project to close. We still have about a week or so left to make the finishing touches, but I think we will be able to implement all of the features by then. The machine learning aspect of the project might not be completely polished by the time we stop making changes to the code, but I expect that we will have it working in some capacity. Once the code has been finished, I plan on going back through everything and separating each piece of the project into separate files, such as the dataset generator and the final version with screen reading implemented. I also want to go through the code and delete anything that did not get used, provide some documentation for certain functions, and generally clean everything up before submitting it. Then, we will be able to direct our full attention to the last few sections of the design document, as well as making finishing touches to anything that we wrote during the design semester. An important part of this process will be making sure that these previous sections are updated to be closer to the process that we ultimately ended up taking, rather than what we had initially thought we were going to do.

Although I haven’t started writing Chapter 5 of the final design document just yet, I have a few ideas as to what I would want to write when I get around to it. I will definitely be talking about the different parts of the project that required us to learn how to do something, such as using certain functions in Python libraries, or figuring out how machine learning could be implemented. Machine learning will most likely be the main topic in the section about future work on the project, since there is likely a lot that we would be able to improve if we continued to work on it. I also plan on restating the main purpose of the project, and how what we were trying to do is something that other recent artificial intelligence projects are also doing to some extent.

Considering that neither of us had much experience with machine learning (and some of the other Python packages that we made use of), I would say that we did a pretty good job following the project outline that we put together last semester. On the other hand, the transition period from Alpha to Beta did not play out exactly as I had envisioned it, since there are pieces from both stages that are still in development, but it should not end up being a big deal since the beginning of the Beta stage did not depend on when the Alpha stage ended. Our development process ended up being more of a back-and-forth type of thing, which was something that I was hoping to avoid, but it was not as much of an issue as I thought it would be. I had mentioned in a previous blog post that I had hoped we could work on certain tasks as a group, and perhaps I should have made more of an effort to implement this change, but the process that had already been established seemed to be working well enough. So, although we had some high and low points this semester, I think we ended up in a good place despite having to learn a few new things along the way.
